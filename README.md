[CleverRaven/Cataclysm-DDA](https://github.com/CleverRaven/Cataclysm-DDA)

的独立分支项目。

---

## 项目概述

大灾变：净化协议（Cataclysm: Cleanwater Bomb）是一个基于 CDDA 实验版的分支项目。

该分支主旨为兼顾真实性与游戏性，在保留Cataclysm系列固有风味的同时实现：

 - 增强现实逻辑一致性
 - 尽可能做到机制合理性与玩家直觉同步
 - 拒绝无明确依据或不合理不公平的削弱
 - 合理平衡真实性与玩家体验
 - 为改动提供退路或他解

本项目的宗旨是在不简化玩法的情况下，尽可能做到不为玩家体验添堵，建立更加符合现实、契合直觉、保留游戏性、对角色与怪物双向且公平的改动机制的改动体系，摒弃真实性必然和游戏性与角色性能冲突的片面观点，做到使开发者能站在玩家的角度思考。

---

## 核心设计原则

1. 现实逻辑一致性
支持有大量参考文献支撑的符合现实逻辑改动，所有机制修改需尽可能符合现实世界物理、生理或工程逻辑，并能够被解释；应当避免任何与现实逻辑出现悖论的改动强行推进。

2. 机制可解释性与直觉可理解性
机制需要是可被理论解释与理解的、符合玩家现实直觉的，在最理想状态下应当使得玩家能够在游戏中使用现实世界可学习的经验直觉进行游戏，而非以超自然知觉逻辑作为系统的主导。

3. 改动公平性
改动应当是公平的、有依据的、非恶意削弱的，例如应当做到对玩家和怪物同时具有的相同功能的削弱不应只针对一方实现，反对单方面或刻意为玩家游玩增加困难及增强怪物的改动。

4. 玩家体验非对抗性
避免因信息不透明或非预期规则导致的无意义失败，明确真实性不等于削弱也不等于缺乏游戏性，改动的同时要兼顾游戏性。

5. 改动内容非极端性
避免一刀切式的禁止行为，避免仅进行删除而不增加代替方案的行为，开发者与贡献者应当为自身做出的减法负责，避免仅为了削弱或增强而进行的极端改动。

6. 渐进式系统演化
所有改动必须模块化推进，避免破坏上游结构与长期可维护性。

---

## 分支边界

为保证长期可维护性，明确以下约束：

- 不以降低难度为目标（改动的结果可以说难度被降低了，但是不能为了降低难度去降低单独，同时也不能为了增加难度而增加难度）  
- 不移除核心生存压力机制  
- 不追求与上游完全兼容  
- 不进行无依据的数值“手感优化”  

核心目标是：

> 在复杂系统中提升机制一致性与可解释性，而非削弱复杂度本身。

## 贡献方式

- 可验证的现实依据
- 对系统性影响的分析
- 可解释性改进方案
